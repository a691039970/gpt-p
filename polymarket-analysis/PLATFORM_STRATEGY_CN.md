# 平台视角任务说明

## 核心转向

本项目不再把观测对象理解为“某个厉害的人”，而是把链上地址当作某个平台信号流的公开出口。

当前临时映射：

- `A = 0x490b4fE78B2FB36f733FeF1b340759e03500eec9`
  - 角色：信号源出口
  - 用途：实时观察平台信号何时出现、出现在哪个盘口
- `B = 0x115edF00e95798fcE4B1c2786942Dc4A5da7f21c`
  - 角色：执行复核地址
  - 用途：验证是否正确跟到了应跟的盘口

## 当前短期目标

1. 当 A 地址发生新交易时，把最新信号实时递送到网页面板
2. 每隔 5 分钟复查 B 地址是否正确跟到了应跟的盘口
3. B 一旦确认已跟，该事项从主窗口隐藏，窗口只保留待处理机会
4. 解释每条“可跟 / 观察 / 不跟”结论背后的规则原因

## 当前中期目标

1. 把项目从“地址跟单面板”升级成“平台信号监控面板”
2. 统一研究 `Z / P / R` 三个平台的相对强弱
3. 不再问“某个人会不会赚钱”，而是问“哪个平台在哪些条件下持续更强”
4. 研究平台间的领先、背离、风格差异和周期切换

## 跟单判断的现阶段含义

- `follow`
  - 当前规则认为这个盘口更接近可复制的有效平台信号
- `observe`
  - 不是说信号错，而是说复制时点不够理想，平台可能还会修正、补仓或对冲
- `skip`
  - 暂不复制，原因通常是品类不在试行范围、价格过低或时机失真过大

## 为什么“平台视角”更重要

个体会变，平台的结构性特征通常更稳定：

- 平台上的信息扩散速度
- 平台擅长的赛事类型和盘口类型
- 平台用户群的风格
- 平台在不同周期中的相对强弱

如果 3 年观察显示 `Z` 长期显著优于 `R`，那么研究重点应是：

- `Z` 在什么条件下持续赢
- `Z` 的优势是不是可复制
- `P` 与 `Z` 是否存在风格差异或轮动关系

## Z / P 研究最低数据要求

每条记录最好至少包含：

- `platform`
- `timestamp`
- `title`
- `slug`
- `outcome`
- `side`
- `price`
- `size`
- `category`
- `event_start_time`
- `transaction_hash`

可选增强字段：

- `result`
- `realized_pnl`
- `unrealized_pnl`
- `closed_at`

1. 面板切换到平台研究 / 执行视角
2. 加 A 的实时更新通道
3. 加 B 的 5 分钟复核状态机
4. 只保留待处理机会，不展示已跟完成项
5. 建立 `Z / P` 统一历史数据结构
6. 开始做平台强弱与周期研究

# Z 权重仓位回测原型

生成时间：2026-03-30

输入数据：
- 年度样本：[z_merged.yearly.current.json](../normalized/z/z_merged.yearly.current.json)
- 权重回测摘要：[z_weighted_backtest.summary.json](../normalized/z_weighted_backtest.summary.json)

## 一句话结论

你刚才指出的问题是对的：

`Z` 不是一个适合固定仓位均匀执行的平台。`

这次把“按样本内结构分数分层配仓”做进回测后，得到的关键发现是：

- 真正赚钱的是最高权重层
- 中权重和低权重层会把利润吃回去
- 所以正确方向不是“所有结构都加权参与”，而是“只允许最高权重层表达，其他层少做甚至不做”

## 1. 这版原型做了什么

和上一版固定仓位不同，这次改成：

1. 先在样本内给每条结构打分
2. 再按分数分层配仓
3. 权重规则：
   - 前 `20%` 结构：`3x`
   - 中间 `30%` 结构：`1.5x`
   - 其余 `50%` 结构：`0.5x`
4. 同时做：
   - 静态样本内 / 样本外切分
   - 逐月滚动样本外验证

## 2. 最重要的结果

### 全结构宇宙

静态样本外：

- ROI：`+0.26%`

滚动样本外：

- ROI：`-1.38%`

但拆层后很关键：

- `3x` 最高权重层：ROI `+1.42%`
- `1.5x` 中权重层：ROI `-3.66%`
- `0.5x` 低权重层：ROI `-4.35%`

### 研究聚焦宇宙

静态样本外：

- ROI：`-2.29%`

滚动样本外：

- ROI：`-0.81%`

同样拆层后：

- `3x` 最高权重层：ROI `+4.38%`
- `1.5x` 中权重层：ROI `-6.19%`
- `0.5x` 低权重层：ROI `-5.08%`

## 3. 这意味着什么

这次结果其实比“固定仓位失败”更有价值，因为它说明了：

1. `Z` 的利润确实不是均匀分布的
2. 高权重层本身是有正 edge 的
3. 真正的问题是我们还在让中低质量层继续参与

也就是说，当前更像：

- `高权重层 = alpha 来源`
- `中低权重层 = alpha 稀释器`

这和你说的“20% 单数贡献 60% 利润”是同一个方向。

## 4. 当前最重要的工作判断

如果把这轮结果压成一句话，那就是：

`问题已经不再是“要不要权重仓位”，而是“除了最高权重层之外，其余层是不是根本不该做”。`

所以接下来最值得验证的不是继续做“全层参与”，而是做：

- `只做 top tier`
- `top tier + 状态过滤`

## 5. 下一步最合理的 V3

建议下一版直接改成：

1. 只保留最高权重层，不做 1.5x 和 0.5x 层
2. 叠加总状态和子策略状态过滤
3. 优先在 `Dota2 / LoL` 结构里验证
4. 核心检验目标改成：
   - 前 `20%` 预测高权重单，是否能贡献大部分样本外利润
   - 去掉中低权重层后，滚动样本外能否转正

## 6. 当前结论

这轮回测已经足够支持一个明确判断：

`Z` 更像“少数高权重机会驱动”的平台，而不是“所有信号按不同权重都能参与”的平台。`

因此，后面的正确方向不是继续优化固定仓位，而是把模型收缩成：

- 只抓最高质量层
- 并且只在正确状态下出手
